Написать поиск как на https://giphy.com/
Документация для апи https://developers.giphy.com/
Как подключаться к апи https://developers.giphy.com/docs/

Что должно быть реализовано:
    1. Поле поиска
    2. Результат поиска должен быть показан в виде сетки внизу от поиска
    3. Можно переключать на результат по гифками или на результат по стикерам. Можно реализовать это в виде табов
    4. Можно сохранять гифки и стикеры. Если есть сохраненные стикеры/гифки, видна ещё одна таба. При переходе на неё будут показаны сохраненные гифки/стикеры
        - После перезагрузки страницы должен быть доступ к сохраненным стикерам/гифкам 
        - Для сохранения можно использовать localStorage
        - Плюсом будет, если вы будете сохранять в indexedDB
    5. Не надо показывать сразу все результаты. Потому что их может быть тысячи. 
    Загрузите сначала 20-30. А когда пользователь проскролит вниз до конца то догрузите ещё.
    Похожий механизм можно наблюдать в ленте новостей вконтакте.
    6. При вводе в инпут для поиска необходимо открывать dropdown с предложенными результатами
        - Предложенные результаты можно сразу добавить в сохраненные
        - При сохранении из предложенных результатах не должен теряться фокус на инпуте для поиска

Что можно использовать:
    1. Любой css фреймворк
    2. Любые апи, которые поддерживает браузер. Советую писать под хром, так как у него всегда самая лучшая поддержка

Что нельзя использовать:
    1. Сторонние js библиотеки/фреймворки
